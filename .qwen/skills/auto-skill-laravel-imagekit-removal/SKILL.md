# Laravel ImageKit Removal

Use this approach when a Laravel project still contains stale ImageKit code after removing ImageKit routes/config, especially when pages fail with `Route [imagekit.auth] not defined.`

## Procedure

1. Do a broad reference scan before editing:
   - Search for `imagekit`, `ImageKit`, `imagekit.auth`, `IMAGEKIT_`, `IMAGE_KIT_`, `ikUpload`, `uploadWithImageKit`, `ik.imagekit`, `new ImageKit`, and `route('imagekit.auth')`.
   - Include `app`, `routes`, `resources`, `config`, migrations, docs, `.env`, `.env.example`, `composer.json`, `composer.lock`, `package.json`, and `package-lock.json`.
   - Also glob for file names containing `ImageKit` or `imagekit`.

2. Fix the root Blade compile/runtime failure first:
   - A Blade expression like `{{ route('imagekit.auth') }}` fails during view rendering/compilation if the route was removed.
   - Remove the Blade route call completely; do not add a dummy route as a workaround.

3. Replace ImageKit JavaScript upload flows with the existing Laravel local upload endpoint:
   - Use a normal file input (`type="file"`, `accept="image/*"`, `multiple` when needed).
   - Validate MIME and max size in JavaScript using the app's global media settings when available.
   - Upload with `fetch('/api/upload', { method: 'POST', headers: { 'X-CSRF-TOKEN': ..., 'Accept': 'application/json' }, body: formData })`.
   - Append `file`, `type`, and `folder` to `FormData`, e.g. `type=products`, `folder=products`.
   - Store returned `data.url` as `image_url` and `data.fileId || data.filePath` as `image_id`.

4. Keep existing product/gallery behavior while swapping storage backend:
   - Preserve arrays such as `productImages` and variant image arrays.
   - Keep existing fields (`image_url`, `image_id`, `alt_text`, `sort_order`, `is_featured`, optional `responsive_urls`).
   - For responsive URL maps previously generated by ImageKit, map all sizes to the local `Storage::url()` URL unless the project has a local thumbnail/resizing service.
   - Re-render the existing UI and reinitialize sortable/drag-drop after uploads.

5. Remove all SDK-specific logic:
   - Delete `typeof ImageKit` checks.
   - Delete auth-parameter fetches.
   - Delete `publicKey`, `urlEndpoint`, `authenticationEndpoint`, token/signature/expire handling.
   - Delete `imagekit.upload(...)` calls.
   - Rename comments, UI labels, and helper names so they no longer mention ImageKit.

6. Clean non-runtime references too if the requirement is zero references:
   - Migrations comments such as `ImageKit file ID` should become neutral (`image_id` only or `storage file ID`).
   - Docs should say Laravel local uploads, not ImageKit/local uploads.
   - Demo URLs like `https://ik.imagekit.io/...` should be replaced with neutral local `/storage/uploads/...` placeholders.
   - Integration tabs or file-manager labels should be renamed to generic `media` / `Media Storage` if they are now local storage UI.

7. Verify dependencies/config are gone:
   - `composer.json` / `composer.lock` should not include ImageKit packages.
   - `package.json` / `package-lock.json` should not include ImageKit packages.
   - `config` should not contain ImageKit config files or keys.
   - `.env` and `.env.example` should not contain `IMAGEKIT_` / `IMAGE_KIT_` variables.

8. Clear and compile caches:
   - Run `php artisan optimize:clear`.
   - Run `php artisan view:clear` and then `php artisan view:cache` to catch remaining Blade compile errors from stale route/component usage.
   - Run `php artisan route:list --path=imagekit`; Laravel should report no matching routes.

9. If `view:cache` finds unrelated missing Blade components, fix the actual component usage instead of ignoring it:
   - Example: replace a missing `<x-frontend.product-card :product="$product" />` with an existing partial include such as `@include('partials.product-card', ['product' => $product])` when that is the project convention.

10. Final verification checklist:
   - Full project grep for ImageKit terms returns zero matches.
   - Product page grep for ImageKit route/SDK calls returns zero matches.
   - Grep confirms the Product upload flow uses `/api/upload` with the correct local folder/type.
   - `php artisan view:cache` passes.
   - `php artisan optimize:clear` passes.
   - PHP syntax checks pass for any edited PHP/migration files.

## Example Local Upload Helper

```js
async function uploadImageLocal(file) {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('type', 'products');
    formData.append('folder', 'products');

    const response = await fetch('/api/upload', {
        method: 'POST',
        headers: {
            'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').content,
            'Accept': 'application/json'
        },
        body: formData
    });

    const data = await response.json();

    if (!response.ok || !data.success) {
        throw new Error(data.message || 'Upload failed');
    }

    return data;
}
```
